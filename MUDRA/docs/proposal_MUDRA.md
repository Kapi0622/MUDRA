## 0. 開発動機

### 0-1. 原体験

NARUTOや呪術廻戦を見ていて、ずっと思っていたことがある。

**「自分も手印を結んで、技を発動させたい」**

画面の中でキャラクターが指を組み合わせ、術式を紡ぐあの瞬間に、子どものころから強く惹かれてきた。コントローラーのボタンではなく、自分の手そのものが「詠唱」になる体験を、ゲームとして実現できないか——それがこのプロジェクトの出発点だ。

### 0-2. タイトル「MUDRA」について

NARUTOの手印のモデルとなった密教・修験道の手の所作も、呪術廻戦が描く術式発動の様式も、その文化的ルーツをたどるとサンスクリット語の一語に行き着く。

**MUDRA（ムドラ）——「手印」を意味する言葉そのものだ。**

あの作品たちが描いてきた「手印で何かが起きる」という感覚の源流にある言葉をタイトルに据えることで、このゲームが単なるオマージュではなく、その本質を直接ゲームプレイとして体験させることへの意志を示したい。

## 1. コンセプト

### 1-1. ワンライン

**「手印を組んで術を放て」——Webカメラで読み取った手の形が、そのまま魔法の詠唱になるアクションゲーム。**

### 1-2. エレベーターピッチ

プレイヤーは「印術師（MUDRA使い）」としてWebカメラの前で実際に手印を組み、術を発動して敵と戦う。
コントローラーではなく自分の手そのものが入力デバイスとなる体験は、既存のゲームにはない「身体ごと没入する」感覚を生み出す。
覚えた印のコンボを咄嗟に繰り出す緊張感と、成功時に画面を覆う派手なエフェクトの爽快感が核となる。

### 1-3. コンセプトキーワード

- **身体入力の没入感** — 手を動かすこと自体がゲームプレイになる
- **記憶と反射の融合** — 印の組み合わせを「覚える→素早く出す」成長実感
- **見た目の映え** — 配信・展示で「何をしているか一目でわかる」視覚的インパクト

### 3-1. 全体フロー

```
[タイトル] → [カメラキャリブレーション] → [印チュートリアル] → [ステージ選択] → [バトル] → [リザルト]
                                                                        ↑                    |
                                                                        └────────────────────┘
```

### 3-2. カメラキャリブレーション（初回 & 任意）

ゲーム開始前に、カメラに手を映して認識精度を確認するステップ。
画面に「手をかざしてください」と表示し、ランドマークが正常に取れていることを
プレイヤー自身が視覚的に確認できるようにする。
認識がうまくいかない場合は、照明や距離のアドバイスを表示する。

### 3-3. 印チュートリアル

初回プレイ時に、基本の手印を1つずつ練習するモード。
画面にお手本の手の形を表示し、プレイヤーがそれを真似る。
成功判定が出たら次の印に進む。全印を練習したらバトルに移行。

## 4. コアメカニクス：手印システム

### 4-1. 手印の種類

印の種類は、実機での認識精度テストを経て確定させる。
**まず3種から始めて精度を確認し、問題がなければ段階的に増やす方針とする。**
片手印と両手印を組み合わせることで識別の幅を確保する。
以下はあくまで設計段階での候補案であり、テスト次第で変更しうる。

**【片手印】候補**

| 印名 | 手の形のイメージ | 属性 |
|------|------------------|------|
| **壱印「開」** | パー（全指開き） | 風 |
| **弐印「握」** | グー（全指閉じ） | 地 |
| **参印「指」** | 人差し指のみ伸ばす | 雷 |
| **肆印「刃」** | チョキ（人差し指+中指） | 水 |
| **伍印「掌」** | 手のひらをカメラに向けて親指だけ折る | 火 |

**【両手印】候補**

| 印名 | 手の形のイメージ | 属性 |
|------|------------------|------|
| **陸印「合」** | 両手合わせ（検出される手が2つ） | 光（特殊） |

> **設計意図:** 各印が「指の開閉パターン」で明確に区別できるように設計。
> MediaPipe のランドマーク21点から指の曲げ角度を算出すれば判定可能な形のみ採用。
> 両手印は「両手が同時に検出されている」こと自体を条件にするため、片手印との混同リスクはほぼゼロ。

### 4-2. 特殊印：発動印・解除印

通常の詠唱印とは別に、専用の役割を持つ特殊印を設ける。

| 印名 | 役割 | 発動条件 |
|------|------|----------|
| **発動印「射」** | 詠唱完了後に術を実際に放つ。これがないと術は発動しない | シーケンス完成後に組む（例：グーで握り込む動作など） |
| **解除印「散」** | 詠唱シーケンスをリセットする。ミスに気づいたとき能動的にキャンセルできる | 詠唱中いつでも組める（例：手をブラブラ振るなど） |

**発動印を設ける理由:**
- 詠唱中の意図しない暴発を防ぐ
- 「溜めて…撃つ！」という意思表示の明確化による気持ちよさの向上
- 将来的に発動印を多様化（「押し出す→直線攻撃」「広げる→範囲攻撃」など）することで、着弾パターンをコントロールする拡張軸として機能しうる（ストレッチゴール）

**解除印を設ける理由:**
- ミスに気づいた瞬間に能動的にリカバリーできる手段を用意することでテンポの崩壊を防ぐ
- 暴発・不発のデメリットは「解除印を使わず、間違ったまま発動印を押した場合」および「時間切れの場合」のみに限定する
- 上級プレイヤーがミス後にすばやく解除→立て直しを行うスキルの天井を作る

### 4-3. 術の発動：シーケンス入力

術は「詠唱印を順番に組んだ後、発動印を組む」ことで発動する。

```
【火球】  伍印「掌」 → 壱印「開」 → 発動印「射」       （2詠唱+発動・初級）
【雷撃】  参印「指」 → 参印「指」 → 壱印「開」 → 発動印「射」  （3詠唱+発動・中級）
【大地震】弐印「握」 → 伍印「掌」 → 弐印「握」 → 発動印「射」  （3詠唱+発動・中級）
```

**入力の流れ：**

1. プレイヤーが詠唱印を形成する
2. 0.4〜0.6秒間ランドマークが安定 → **「印確定」** の演出（光る＋SE）
3. 次の詠唱印 or 発動印を待つ（制限時間あり）
4. 発動印を確定 → **術発動エフェクト＋ダメージ処理**
5. ミスに気づいた場合 → **解除印「散」** でシーケンスをキャンセルしてやり直せる
6. 間違ったまま発動印を押す or 制限時間切れ → **失敗（暴発 or 不発）**

### 4-4. 発動タイミングとリスク・リターン

| 要素 | 内容 |
|------|------|
| **印の制限時間** | 印と印の間は 2.0秒以内に次の印を確定させる |
| **術の威力** | 印の数が多いほど高威力 |
| **暴発（ミス）** | 間違ったまま発動印を押す or 時間切れ → プレイヤー自身にダメージ |
| **速度ボーナス** | 制限時間の半分以内で全印を完了 → 威力1.5倍 |

## 5. バトルシステム

### 5-1. 基本構造

**ボス戦特化のシングルプレイ。** プレイヤーは画面の手前側、ボスは画面奥に配置。

```
┌──────────────────────────────┐
│         [ボスHP]              │
│                              │
│        【ボスキャラ】          │
│                              │
│   ← ボスの攻撃エフェクト →    │
│                              │
│  ┌──────┐  ┌──────────────┐  │
│  │カメラ │  │ 印シーケンス  │  │
│  │プレビュ│  │ 表示エリア   │  │
│  └──────┘  └──────────────┘  │
│  [プレイヤーHP]  [術リスト]    │
└──────────────────────────────┘
```

### 5-2. ボスの行動パターン

ボスは一定間隔で攻撃を行い続けるリアルタイム構成をベースとしつつ、
攻撃前の予告演出でプレイヤーに対応猶予を与えることでターン制的なテンポを生み出す。

```
【ボスの行動サイクル（繰り返し）】
  攻撃予告（1.5秒の予告演出）
    → 通常攻撃：HPでそのまま受けて詠唱を続行するのが基本スタイル
    → 大技 or 厄介な攻撃：防御印（弐印「握」）を構えてダメージ軽減
    ※ どちらを選ぶかはプレイヤーが判断する
```

**被弾仕様:**
- 被弾してもシーケンスはリセットされない（詠唱を続行できる）
- 防御印の価値は「大技に対する軽減率が大きい」ことで担保する
- 基本ダメージは受けながら攻撃し続け、大技・厄介な攻撃のみ防御印を選択するのが基本のゲームプレイリズム

### 5-3. 術の個性軸（属性一辺倒を防ぐバランス設計）

属性相性だけでは弱点属性の術を連打するゲームになるため、
術には属性以外の個性軸を持たせてバランスを取る。

| 個性軸 | 例 |
|--------|-----|
| **射程・範囲** | 単体狙い撃ち / 全体範囲 / 直線貫通 |
| **ヒット数** | 単発高火力 / 多段低火力 |
| **発生速度** | 即発 / ため（発動まで遅延あり・威力大） |
| **副次効果** | スロウ・スタン・DoT（継続ダメージ） など |
| **詠唱コスト** | 印数が多いほど強力だが時間がかかる |

> **バランス方針:** ボスのフェーズや行動パターンによって「最適な術の種類」が変わるよう設計する。
> 弱点属性でも詠唱コストが高ければリスクが生じるため、属性・個性・状況の三つ巴で判断を迫る。
> 具体的な数値バランスはプロトタイプでの検証を経て調整する。

### 5-4. ダメージ計算の要素

- **術の基礎威力**（印の数に比例）
- **属性相性**（ボスごとに弱点属性がある）
- **速度ボーナス**（素早く印を組めた場合）
- **コンボボーナス**（連続で術を成功させると倍率UP）

### 5-5. ボスの種類（ステージ案）

| ステージ | ボス | 弱点 | 特徴 |
|---------|------|------|------|
| 1 | 泥人形 | 風 | 攻撃が遅い。チュートリアル的 |
| 2 | 水蛇 | 雷 | 連続攻撃が多い。防御判断が重要 |
| 3 | 炎魔 | 水 | 攻撃チャンスが短い。速度が問われる |
| 4 | 影龍 | 光 | 両手印（陸印）が必要。総合力勝負 |

## 6. UI/UX設計方針

### 6-1. カメラプレビュー

画面の左下にプレイヤーのカメラ映像を小窓で常時表示。
ランドマークをオーバーレイ描画して「認識されている」ことを視覚フィードバック。

### 6-2. 印シーケンス表示

画面の右下に「次に組むべき印」をアイコンで表示（術の詠唱ガイド）。
初心者モードではガイドあり、上級者モードではガイドなし（暗記が必要）。

### 6-3. 印確定の演出

印が認識された瞬間に以下のフィードバックを返す。
- 手のランドマーク周辺が光るエフェクト
- 確定SE（「パシッ」という音）
- 画面の印アイコンがチェックマークに変化
- 成功時の短い画面フラッシュ

### 6-4. 術発動の演出

発動印を確定した瞬間に 1〜2秒のカットイン演出。
術名が画面中央にテロップ表示 → エフェクトがボスに着弾。
ここの演出の気持ちよさがゲーム全体の満足度を左右する最重要ポイント。

## 7. 世界観・ビジュアル方向性

### 7-1. 世界観

和風ファンタジーをベースとしつつ、現代的なデジタル感を加えたハイブリッドスタイル。
「印術」は古代から伝わる秘術で、現代に生きる印術師が封印された魔物と戦う設定。

### 7-2. ビジュアルスタイル：**B. ドット絵 + エフェクト（確定）**

キャラクター・背景はドット絵で制作し、術エフェクトはパーティクルで3D的に派手に出す。
コントラストによりエフェクトの映えを最大化する。

**3D空間を使う場合はHD-2D的アプローチを採用する。**
- 3D空間にドット絵スプライトを配置することで、モデリングコストを抑えつつ奥行きのある画面を実現
- 敵・NPC・UI素材は基本的に2Dドット絵で統一
- モデリング素材の用意は極力避ける

> **WebGLビルドの注意:** VFX Graph は現時点でWebGLサポートに制約があるため、
> エフェクトは従来の Particle System（Shuriken）ベースで実装するか、
> LitMotion 等でスプライトアニメーションとして演出する方針を技術選定フェーズで確認する。

## 8. 差別化ポイントと強み

### 8-1. 競合との差別化

| 比較対象 | 本作との違い |
|----------|-------------|
| NARUTOゲーム | 印はあくまで演出。本作は印そのものがゲームプレイ |
| リズムゲーム | 音楽に合わせるのではなく、戦況判断で印を選ぶ戦略性 |
| ジェスチャー系ゲーム（Kinect等） | 大きな体の動きではなく、手先の繊細な形の認識 |
| VRハンドトラッキングゲーム | Webカメラだけで遊べる手軽さ。VR機器不要 |

### 8-2. ポートフォリオとしての強み

- **技術的アピール:** MediaPipe + Unity の統合、リアルタイムML活用
- **設計力のアピール:** 入力→判定→演出のパイプライン設計
- **企画力のアピール:** 「手印」という身体入力をゲームメカニクスに昇華した発想
